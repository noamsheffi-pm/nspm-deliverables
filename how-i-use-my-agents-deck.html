The whole idea, in one line

  • Everything I work on is a plain text file in one folder on my laptop
  • My agents read those files and write back to them
  • I stay the approver. Nothing happens that I did not say yes to
No new app to buy. Just files an agent can read and write, and a human who decides. If you remember one thing, remember this.
This is the whole trick. No new app. Files agents can read and write, and a human who says yes or no. The folder is an Obsidian vault, which is just markdown files.

Same six codes on every agent's brief. I sweep the whole stack like a card deck in about ten minutes.
This is why it scales. I never learn a new interface per agent. Same six codes everywhere, swept over coffee. The "n" codes matter later, they become training data.

They earn trust one rung at a time

1
Look, don't touch
Watches and reports. No writing anything.
2
Approved writes
Can act, but every batch waits behind my yes.
3
Runs while I sleep
Scheduled on its own, still inside the approval gate.
No promotion on paper. An agent moves up only after I have watched it do the new thing once. Cappy moved to rung 2 only after its first real write was applied and I checked it.
This is what makes unattended running safe. A new agent cannot write anything. It gets a privilege only after one supervised run proves it. Trust is granted, not assumed.

They grade themselves

  • Once a week, each agent compares what it proposed against how I corrected it
  • My n codes are the answer key
  • It then drafts an edit to its own instruction file
  • The edit lands only when I say yes
My corrections are not lost. They are the training signal. Next week the agent is a little better, without me rebuilding it.
This is the compounding part. Corrections become the training signal. The agent proposes a fix to its own prompt, I approve it, next week it is better.
