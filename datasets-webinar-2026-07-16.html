Part one // what goes wrong

One table, six problems

PERMISSIONS

All or nothing

You can't manage permissions on some of the data.

PERFORMANCE

Scans everything

Every query scans everything, greatly impacting performance.

SCHEMA

One messy schema

The table holds every kind of data at once.

RESOURCES

No prioritization

Can't prioritize resources for certain tables over others.

DISCOVERY

Not discoverable

Data types are just embedded rows in one table.

DATA TYPES

One type only

Only one type of data — timeseries or lookup, but not both.

No one would run a production database like this
Part one // agents accelerate every problem

Agents accelerate all six problems

PERMISSIONS

Now autonomous

Can't scope an agent to part of the data.
All-or-nothing access, unsupervised.

PERFORMANCE

Many queries, not one

Every exploratory step scans everything.
An agent fires dozens.

SCHEMA

More to misread

The agent reasons over one giant schema.
More to read, more to get wrong.

RESOURCES

Runaway load

No prioritization.
A runaway agent starves the queries that matter.

DISCOVERY

Blind guessing

Nothing is discoverable, so the agent guesses.
Wildcard searches, wrong turns.

DATA TYPES

Can't tell them apart

Timeseries and lookup mixed together, so the agent mixes them up.

A person with a dashboard knows where to look. An agent without boundaries searches everything, and every wrong turn is a Token Tax.

Dataspaces and Datasets

The solution // like a database, like a table

Dataspaces and Datasets

Dataspaces · like a database
Logical containers
Structured containers that provide organization — the centralized control layer for the datasets inside them.
RetentionPermissions
Delivers: governance, attribution, compliance
Datasets · like a table
Related data collections
Tightly scoped data within a dataspace — segmented by team, service, or environment.
StorageSchemaLifecycleQuota
Delivers: clarity, segmentation, isolation
The solution // two kinds of dataset

Streaming and Summary datasets

Streaming dataset
Populated by routing
Data is routed straight in, in real time, as it streams into the platform.
Summary dataset
Populated by a Background Query
An ad-hoc / scheduled DataPrime query writes its results into it.
