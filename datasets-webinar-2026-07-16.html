Part one // what goes wrong

One table, six problems

PERMISSIONS

All or nothing

You can't manage permissions on only some of the data.

PERFORMANCE

Scans everything

Every query scans everything, greatly impacting performance.

SCHEMA

One messy schema

The table holds all data types at once, causing schema collisions and union types.

RESOURCES

No prioritization

Can't prioritize resources for important data over others.

DISCOVERY

Autocomplete mess

Field discovery is messy and cumbersome, impairing user experience.

HIGH COST

Wasted resources and money

Finding a needle in a haystack is slow and costly, eventually increasing MTTR.

No one would run a production database like this

The solution // two kinds of datasets

Streaming and Summary datasets

Streaming dataset
Raw data populated by routing
Data is routed straight in, in real time, as it streams into the platform.
Summary dataset
Results populated by a Background Query
Pre-aggregate results for long-term trends, re-use in dashboards, or as part of your data lake.

The solution // self-serve everything

The system dataspace

Internal metadata Coralogix maintains about your own environment — schema evolution, alert activity, usage, and audit — queryable like any other dataset with source system/…
aaa.audit_events

A complete trail of who did what, and when, across Coralogix.

alerts.history

The full lifecycle of every alert — triggers, status, priority over time.

engine.queries

Every query's execution details, to find slow queries and diagnose failures.

engine.schema_fields

How each dataset's schema evolves over time, for debugging and governance.

dataplan.usage_events

Your team's ingestion, unit by unit, after ratios are applied.

dataplan.quota_events

Quota changes, threshold breaches, blocks, and resets as they happen.

labs.limit_violations

Each time a configured limit is exceeded — threshold vs. observed value.

notification.deliveries

Notification Center deliveries — successes and failures, for routing audits.

Query any of them like any other dataset — Explore, dashboards, automated reports
