You own it

A local folder of Markdown. No account, no vendor, no internet required.

Not another app to manage. Obsidian is just a folder of text files with a good UI on top. If you stop using it, your files are still there — readable in any editor, forever.
Local means you need a backup. Since the vault lives on your machine, losing your laptop = losing your notes. Noam's solution: put the vault folder inside Google Drive. Drive syncs it to the cloud automatically, zero config. Create your vault at ~/Google Drive/My Drive/Obsidian/ and you're covered from day one.

Cmd+. opens the Properties panel — fill frontmatter visually without writing YAML manually.

Tags — always at the end of the line

- Review header treatment in the modal#Task
- Waiting on eng answer on hover state#follow-up
- Decision: 16px base font everywhere#decision
- Empty state UX is unclear#open-question
- This section needs a Figma frame link#figma-needed
Position matters. Tags must be at the end of the line. Noam's vault tools extract data by tag position. Always.

Shell Commands

One keystroke — Claude runs on your note

The Shell Commands plugin bridges Obsidian and the terminal. You register a shell script as a named Obsidian command, bind it to a hotkey, and it runs against whatever note is open — passing the file path automatically via {{file_path:absolute}}.

How Noam uses it
Cmd+Alt+G
hotkey in Obsidian
→
grammarly.sh
shell script fires
→
claude -p ...
Claude Code runs
→
note updated
Obsidian reloads
The script pattern
#!/bin/zsh
# Called from Shell Commands plugin — Obsidian passes the file path
FILE_PATH=$(echo "$1" | sed 's/\\//g')

# Build prompt from a Claude command file, inject the file path
TMPFILE=$(mktemp)
sed 's|\$ARGUMENTS|'"$FILE_PATH"'|g' ~/.claude/commands/grammarly.md > "$TMPFILE"

# Run Claude non-interactively on the note
cat "$TMPFILE" | claude -p --allowedTools "Edit,Read,Write"

rm -f "$TMPFILE"
Register a new command — 3 steps
1

Add the command

Settings → Shell Commands → + New shell command
Command: /path/to/your/script.sh "{{file_path:absolute}}"
Alias: anything — this is what appears in the command palette.

2

Bind a hotkey

Settings → Hotkeys → search Execute: {your alias} → click + and press your key combo.

3

Use it

Open any note, press the hotkey. The script receives the file path, runs Claude on it, and Obsidian picks up the changes. No terminal switching needed.
