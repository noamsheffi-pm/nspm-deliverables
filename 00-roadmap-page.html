Cross-agent plumbing wiring + join

Two distinct mechanisms wire the family together. Wiring = typed handoff Agent A → Agent B with action contract. Join = read-only aggregator across all agents into one morning view. Don't conflate.

Wiring (A → B)
Point-to-point. Agent A emits typed signal to Automation/Agents/Wiring/{from}-to-{to}.md; Agent B drains and acts. Stable dedup_key + kill switch (enabled: false).
Spec: Automation/Agents/cross-agent-wiring-spec.md
Join (aggregator)
Many-to-one read. /cross-agent-join correlates today's per-agent files via 5 rules → cross-agent.md. Called by /today Step 1g. Canonical morning read.
Skill: System/Agents/Claude/commands/cross-agent-join.md

Operating principles L1 → L13

L1
Inherit the proven shape
Yoshi's wrapper, plist, mcp-config-empty, state-file pattern, JSONL token logging, atomic writes. Copy and adjust.
L2
Absolute paths in headless
claude -p doesn't propagate cwd. Every path starts with /Users/noam.sheffi/CX/.
L3
Token tracking from day 1
JSONL to Automation/{Agent}/usage.log. Without it, you can't tell when an agent gets expensive.
L4
First-tick state seeding
On first sight: set last_seen_ts = current latest_ts. Don't classify pre-existing backlog as new on heartbeat 1.
L5
DM-to-self target
Slack user ID U080G5KHZQB, not hardcoded D… channel ID.
L6
Quiet hours and weekends
20:00–08:00 Lisbon and Sat/Sun: state still updates, only DMs silenced.
L7
Atomic state writes
Temp file + rename. Never partial. State corruption = false positives next tick.
L8
Memory-aware filter
Every patch through memory rules: no em dashes, no "align with Aviv", no other-team estimates, plain action items, "our portfolio" voice.
L9
One-key approval gate
Never write directly to canonical artifacts. Inbox + y approval — always.
L10
Kill criteria in the plan
Approval <50% any day = freeze. Inbox flood >N = tighten. Filter rewrites >50% = iterate. Lock BEFORE soak.
L11
One agent per loop
Don't merge agents to save boilerplate. Independent state + schedule + kill switch > DRY.
L12
Concept → plan → build → soak → memory
Every agent passes through these gates. Never skip soak; never write memory before soak passes.
L13
Model selection by cadence
Haiku 4.5 for 2h loops · Sonnet 4.6 default for scheduled · Opus 4.7 only for one-shot reasoning skills.
