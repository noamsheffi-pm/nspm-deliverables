§1 Anatomy of "Last run per agent"

The most important section. One row per agent, latest run only. Read left to right — the "On me" column is what matters.

Agent Status Last run When Took On me KEY Summary

On me

Items waiting on you right now. If 0, ignore this agent today. If non-zero, click "Last run" to see what.

Status

One glance at the icon column tells you health. ✅ = ignore. Anything else = look.

Last run (link)

Click to open the per-run detail file. Body explains what each watched item is, who's expected to reply, what's awaiting action.

When + Took

"When" trending old = agent didn't fire. "Took" trending up over weeks = something's getting bloated.

The other three sections

Each one zooms in on a different failure mode. You'll rarely need them — §1 is the canary.

§2

Pending — waiting on me

Sums "On me" across all agents and links to the source. If §1 shows 0 across the board, this section is empty. When it has rows, click [[yoshi-inbox]] or [[luigi-state]] to act.

§3

Failures (last 7d)

Anything not ✅ ok in the last week. Errors, warnings, skipped runs. Empty = healthy. Lights up only when an agent crashed, hit MCP outage, or tripped a guard rail.

§4

Activity (30d)

Per-agent rollup: runs · processed · closed · pushed-to-me · success%. Are they earning their keep? High "Pushed to me" relative to "Processed" = good filtering. Low success% = needs attention.

Status legend

Five values an agent run can land on. Memorize the first two — the rest are rare.

StatusMeaningAction
✅ ok Ran clean. No errors. Nothing — this is the goal state.
⚠️ stale Ran ok, but missed its next scheduled tick by >1h. Check launchd / scheduler. Agent may not be firing.
🟡 warn Ran with anomalies — partial walk, capped items, ambiguous classifier. Open run file, read body, decide if it's recoverable.
❌ fail Threw or aborted partway. State may be inconsistent. Check errors, fix root cause, re-run with --force.
⏸ skipped Guard rail fired — quiet hours, session-skip <4h, MCP down. Usually fine. Skipped is healthy on weekends.

The 60-second morning routine

Four steps. If steps 2 and 3 both come back clean, you're done in 10 seconds.

1

Open the dashboard

Cmd-O → "Agent Control Room". Dataview re-renders on view, so it's always fresh.

2

Glance §1 "On me"

Any non-zero number? Click that agent's "Last run" link to see what's waiting.

3

Check §1 Status

Anything not ✅? Drop into §3 for the failure detail. Most days: nothing here.

4

Done

Close the file. Trust the agents to wake you up only when there's signal.

Adding a new agent (Mario, future)

🐢

Mario joins the dashboard automatically

When you build Mario: add a final step in his skill that writes one file to Automation/Agent-Runs/{YYYY}/{MM}/mario-<ts>.md using the schema in Automation/Agent-Runs/README.md. No dashboard changes needed. Dataview sees the new agent: mario value and a Mario row appears in §1, §2, §3, §4 the next time the dashboard renders. Same pattern for every agent after that.
