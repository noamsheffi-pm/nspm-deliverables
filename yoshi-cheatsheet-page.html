1 Your only manual step

Paste this line in any vault file after sending a Slack ask. Get the permalink from Slack: right-click message → Copy link to message. 

- [ ] {1-line context}: {permalink} #slack-track #{Owner}

Tag taxonomy

#slack-track #slack-track-watch #slack-track-closed #follow-up
TagWhen to useYoshi behavior
#slack-track Active ask — you want a reply DMs on substantive reply + bumps after 3 days stale
#slack-track-watch Passive watch (FYI threads) DMs on any reply, never bumps
#slack-track-closed Auto-set after sweep Stops watching
#follow-up NON-Slack tasks Yoshi never touches this
Format flexibility
Only the tag and permalink are parsed. Free-form prose around them is fine. The #Owner tag is recommended (used in DM attribution) but not required.

2 What Yoshi runs and when

/yoshi-graze 23:00 nightly
Scans last 24h from:@me Slack messages, proposes new #slack-track entries → yoshi-inbox.md. Catches asks you forgot to tag.
/yoshi-watch every 2h, 09–19 weekdays
Polls every tagged thread, DMs you on substantive replies, fires one-time bump on 3-day stale items. The only thing that pings.
/yoshi manual / from /today
Closes resolved items, writes reply summary into source file, files inbox entries. Read-only on Slack — no DMs from sweep.
Quiet hours
20:00–08:00 Lisbon and weekends → no DMs. State still updates so nothing is lost; alerts deferred to the next active tick.

6 Substantive vs trivial classifier

Substantive (DM fires)

  • More than 2 sentences
  • Contains I think, we should, concern, issue, but
  • References RFC, CX-NNNN, file path
  • Raises a clarifying question

Trivial (state updates, no DM)

  • 👍 / "ok" / "thanks" / "ack"
  • Single short sentence with no signal words
  • Emoji-only reactions

7 Force a reset or close manually

Want toEdit
Close item without waiting for sweep Change #slack-track → #slack-track-closed on the source line
Re-fire stale nudge that already fired yoshi-state.json → set stale_pushed: false on key
Stop tracking entirely Remove #slack-track* tag OR delete state entry
Force full re-poll (ignore state) Run /yoshi --full

8 Hard separations

Don't get confused
  • • #slack-track* is owned by Yoshi only
  • • #follow-up is for non-Slack tasks — Yoshi never touches it
  • • /yoshi-watch is the only thing that sends DMs
  • • /yoshi (sweep) is read-only on Slack, only edits vault
