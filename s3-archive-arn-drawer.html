🔐 How Coralogix accesses your bucket 2

You create a dedicated IAM role that only your Coralogix account can assume, locked to an External ID you choose. You can revoke or rotate it anytime, and every change is audited. Coralogix never uses a shared credential to reach your data.

✨ Your bucket is set up with the basic access method (shared role coralogix-archive-eu1). A new, more secure setup is now available — a dedicated role locked to your own External ID. Add it alongside your current setup (no forced cutover), then remove the old access once migration completes below.

External ID *
Any string you choose. It must match the value in the trust policy.
Role trust policy 4
{
  "Version": "2012-10-17",
  "Statement": [{
    "Sid": "CoralogixAssumeRoleWithExternalId",
    "Effect": "Allow",
    "Principal": { "AWS": "arn:aws:iam::625240141681:role/coralogix-archive-eu1" },
    "Action": "sts:AssumeRole",
    "Condition": { "StringEquals": { "sts:ExternalId": "a48191a0-0e5d-4b61-a062-134484ee8fb5" } }
  }]
}
