Governance · Schema

Govern Your Schema on a Cadence

Schema Explorer — Monthly Reports

Schema Explorer now produces a recurring monthly report inside the product, with configurable retention. Review schema sprawl and data growth on a schedule instead of discovering field explosion and cost spikes during an incident.

Monthly Report, In-Product

A recurring monthly snapshot of your schema lives in Schema Explorer — no export pipeline, no manual pull.

Configurable Retention

Choose how long reports are kept, so you can see how schema and data growth trended over months.

Catch Sprawl Before It Costs

Field explosion and growth hotspots surface on a schedule — governance becomes proactive, not incident-driven.

Foundation for Cost Reviews

The recurring report is the surface monthly cost and governance reviews are built on.

Governance · Quota

Quota Limits for System & Default Datasets

System & Default Datasets Quota Limits

Reserved-capacity quota limits for both system/ and default/ datasets are now default-on for everyone — the feature flag has been removed. These datasets get treated as first-class governed entities with their own allocation, no longer a whitelisted preview.

Default-On for All Tenants

Feature flag removed — quota limits for system/ and default/ datasets ship to everyone.

Both Dataspaces Governed

System and default datasets each get their own allocation boundary, governed like any other dataset.

Cap User-Defined Datasets

The default/ dataspace caps user-defined datasets, with a usage chip showing where you stand against the limit.

Sets Up Unit-Based Locks

The allocation model the upcoming unit-based reservations build on.

Monetization · Data Usage

Read Usage the Way You Manage It

Data Usage — User-Controlled Aggregation

Choose how usage is aggregated — Auto, Daily, Weekly or Monthly — across Data Usage and the Quota Manager, so the view matches the budget cycle you actually plan against. Daily noise when you're planning a monthly budget is gone.

Four Aggregation Modes

Auto, Daily, Weekly, Monthly — applied consistently across Data Usage and the Quota Manager.

Match Your Planning Horizon

Read usage at the grain you budget by, instead of squinting at daily bars.

Consistent Across Surfaces

The same aggregation control on both the usage and quota views.

Cleaner Trend Reading

Smoother weekly/monthly rollups make month-over-month trends obvious.

Monetization · Data Usage

Trustworthy Org-Wide Usage Visibility

Org Quota Manager — Sharper Usage View

See exactly which teams are heading for a block, and get org totals you can trust — internal audit teams no longer skew the numbers you report up to finance.

Block-Visibility Columns

New block-days and estimated-blocked-units columns make pressure visible before data starts dropping.

Max-Usage Progress Bar

A max-usage bar with numeric sort surfaces the teams closest to their limit.

Exclude _audit Teams

Internal audit teams are excluded from org-level counters, so the reported total is the real one.

Cleaner Olly Usage View

Olly usage reads correctly for paid accounts and no longer shows zero when units are reserved by a quota rule.

Monetization · Data Usage

Usage Reporting for Pay-As-You-Go

PAYG Usage Filter & Report Export

Pay-as-you-go customers can now filter usage and export a dedicated PAYG report — the same export experience the rest of the monetization surface offers, now for the PAYG model.

Dedicated PAYG Report Type

Export a report scoped to the pay-as-you-go usage model, not a generic dump.

Filter Before You Export

Narrow PAYG usage to what matters before pulling the report.
